今日のリアルな結論

正直、ChatGPTは解約しようかなと思っていたけど、 Codexがあるなら継続する価値は十分あると感じました。

難しそうな“AIエージェント”を、ちゃんと 「自分も触れそう」な距離まで下げてくれるのが良かったです。

おけもんがAIエージェントに惹かれる理由

もちろんコードを書いてアプリやツールを作るのも面白い。けれど一番ワクワクするのは、 「自分の庭が育っていくこと」です。

AIエージェントは、
“自分の庭”を育てる存在

おけもんにとってAIエージェントの面白さは、単に作業を早くすることではありません。 自分のフォルダを読み込み、これまでの創作物や思考やメモを踏まえて返してくれて、 さらに出来上がったものをまた庭に戻していけるところです。

だから感覚としては、ただの便利ツールというより「あつ森を育てていく感じ」に近い。 置いたものが増えるほど、島に文脈が生まれ、次にできることも広がっていきます。

ためるメモ・創作物・会話を置く
読ませるフォルダごと文脈を渡す
一緒に作る下書き・整理・発想を進める
戻す成果物をまた庭に植える

なぜ今これが大事なのか？

これは「おけもんが楽しいから」だけではなく、 これから先のAI時代においてかなり本質的な動きだと思っています。

1. 文脈が資産になる
その場限りの会話より、自分の中に蓄積されたナレッジの方が、あとから何度も使える。
2. AIが自分仕様に育つ
フォルダ内の思考や創作物が増えるほど、返ってくる提案も自分に合ったものになっていく。
3. 出力が消えずに残る
AIとのやりとりで生まれた成果物をまた庭に戻せるから、毎回ゼロから始まらない。
4. 未来の差になる
これから重要になるのは、誰が一番多くAIを使ったかより、誰が一番自分の知識と文脈を育ててきたか。

まずここが違う
普通のChatGPT vs AIエージェント

イメージしづらい人向けに、ざっくり役割で分けると分かりやすいです。

いつものAI

普通のChatGPT

基本は「質問する → 答えが返ってくる」の往復。相談相手としてはとても優秀です。

1
聞く
分からないことを質問する
2
返ってくる
説明、案、文章、アイデアが返る
3
人が動く
その先の実行や整理は自分で進める
一歩先のAI

AIエージェント

会話しながら、調べる・整理する・作る・直すなど、作業の流れまで前に進めやすいのが特徴です。

1
頼む
「こうしてほしい」をまとめて渡す
2
考えながら進む
必要な手順を組み立てて処理する
3
途中までやってくれる
たたき台、調査、実務寄りのアウトプットまで届きやすい

ポイントはここ

AIエージェントは「別物の難しいツール」というより、 “答えて終わり”から“作業を前に進める”へ広がったAIと捉えるとかなり分かりやすいです。 その入口として、Codexはかなり優秀でした。

時間が経つほど効いてくる投資でもある

AIエージェント活用は、最初から爆発的な成果が出るというより、 ナレッジがたまるほど価値が増していく“複利型”の取り組みに近いです。

1ヶ月後

まだ手応えは薄いけど、種は見え始める

最初は「これ意味あるのかな？」となりやすい時期。でも日記、音声ログ、メモが少しずつたまり、AIとの対話も自分っぽくなり始める。

この段階では、効果よりも“残す習慣”を作ることがいちばん大事。

3ヶ月後

過去ログが効き始める

似たテーマをAIが拾いやすくなり、「前にもこう言ってたよね」が起きてくる。1つの原液から複数媒体へ変換する流れも回しやすくなる。

このあたりで“庭が育ってきた感覚”が出始める。

1年後

自分仕様の土台ができる

思考の癖、価値観、言い回し、よく出る問いがたまり、AIがかなり自分仕様になる。発信、商品、共有資料もゼロから作らなくてよくなる。

「資産がある人のAI活用」に入っていくタイミング。

3年後

差がはっきり出る

その場の便利さだけでAIを使ってきた人と、自分の文脈を育ててきた人で、出てくる質と速さに大きな差がつく。

ここではもう、AIを使うこと自体ではなく、何を積み上げてきたかが武器になる。

こんな“無限ループ”も作れるかも

今日読み込んだ「好き勝手に書いたノートをAIで広げていく永久機関」の話を、おけもん流の庭づくりに重ねると、こんな活用イメージが見えてきます。

1つのメモから、次の発信がどんどん生まれる

起点はあくまで自分の一次情報です。AIに最初から全部書かせるのではなく、 自分のノートや音声ログや壁打ちを種にして、AIが編集者として広げていくイメージです。

1. まず書く / 話す
デイリーログ、音声ログ、セッションの気づきをそのまま庭に置く。
2. AIが問いを出す
「この話の前提は？」「読者が気になるのは？」をAIエージェントに掘ってもらう。
3. 対話で深める
AIとの壁打ちや人との会話で、もとのメモをさらに深くする。
4. 媒体を変える
note下書き、X長文、Chatwork共有文、音声台本などに展開する。
5. できたものを庭に戻す
完成した文章や図解や会話ログをまたObsidianに入れて、次の材料にする。

おけもんなら、たとえばこう使える

今日の動画要約の文脈を借りるなら、 「好き勝手に出した一次表現を、AIが届け方へ変換してくれる」という流れです。

例1. 音声ログ → note下書き
朝の壁打ち音声から、AIがテーマを抜き出し、読みやすい文章に整える。
例2. note → X長文 / 共有文
もとの思想は変えずに、SNSやChatwork向けの届く形へ編集する。
例3. 会話 → 次の問い
AIが「このテーマ、次はここを深めると面白い」と新しい問いを返してくれる。
例4. 成果物 → 再び庭へ
できた図解や投稿をまた保存することで、次回はもっと自分っぽい提案が返ってくる。

Codexのどこが良かったの？

なかじさんの動画エッセンスと、今日1日触った所感を重ねると、強みはこの5つに集約されます。

entry

1. ほぼChatGPT感覚で触れる

これが一番大きいです。AIエージェントと聞くと身構えるけど、 Codexは入口の感覚がかなり素直。だから「ちょっと試してみよう」が起こりやすいです。

quality

2. GPT-5.4でクオリティが高め

ただ簡単なだけじゃなく、今のバージョンになってから “ちゃんと使える感”がかなりある。入口だけ優しいツールでは終わらない印象でした。

cost

3. 上限が来づらいのが強い

なかじさんの動画でもここは大きな論点でしたが、 実際に使うと「まだいける」安心感がかなり大きいです。気軽に試行錯誤しやすいのは本当に助かります。

beginner

4. 初心者の入口として“あり”

ターミナルや設定で最初につまずきやすい人でも、 Codexなら「AIエージェントってこういうことか」が体感しやすい。最初の1本としてかなり優秀です。

decision

5. 継続する理由になる

「解約しようかな」が「続けようかな」に変わったのは、 それだけ実用の手応えがあったということ。触って初めて良さが分かるタイプの強さがありました。

Claude Code

作り込みや深い開発向け。ターミナル前提の場面もあり、慣れると強いけれど、最初の摩擦はやや大きい。

動画ではClaude Code側に軍配。ただし、そこまで踏み込まない人にとってはCodexの気軽さがかなり魅力。

D

結論

どちらが絶対上ではなく、「自分にとって入りやすい方から始める」が正解。その意味でCodexはかなり強い入口。

初心者にとっての本音

「すごいのは分かるけど、 それ、私にも触れるの？」

Codexが良いのは、この不安をかなり減らしてくれるところ。 いきなり“ガチ勢の世界”に入る感じではなく、いつものChatGPTの延長線で、一歩先を体験できるのが大きいです。

じゃあ、どんな人にCodexが向いてる？

「AIは少し触ってる。でもAIエージェントはまだピンと来てない」という人ほど、相性がいいと思います。

for beginner

まず入口を知りたい人

AIエージェントの世界を、難しすぎず体感したい人向け。

  • ChatGPTは使ったことがある
  • でもターミナルはちょっと怖い
  • まず「何ができるの？」を掴みたい
for worker

日常実務に混ぜたい人

壁打ちだけでなく、調査や下書き、整理も少し前に進めたい人向け。

  • 調べものをまとめたい
  • たたき台を速く作りたい
  • 手数を増やしたい
for explorer

ワクワクしながら試したい人

理屈より先に、まず触って「おもしろい」を感じたい人向け。

  • 難しい設定は後回しでいい
  • まず体験してから判断したい
  • 上限を気にしすぎず試したい
